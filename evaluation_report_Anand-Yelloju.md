## Student Summary
The RAG pipeline was implemented using LangChain with FAISS as the vector store and MiniLM (sentence-transformers/all-MiniLM-L6-v2) for embeddings. The markdown documents were ingested and chunked using RecursiveCharacterTextSplitter with a chunk size of 1000 and overlap of 200. The retriever fetches the top-k relevant chunks and the response is generated using a Groq-hosted Llama 3.1 model.

The evaluation was performed using RAGAS metrics including Faithfulness, Answer Correctness, Context Precision, and Context Recall. The system achieved high faithfulness and perfect context recall, indicating that retrieved context was relevant and answers were grounded in the documents. Lower context precision suggests that additional chunks were retrieved beyond what was strictly necessary. Answer correctness was moderate due to differences in phrasing compared to the ground truth answers.

Overall, the RAG system successfully answered domain-specific PMKVY questions, avoided hallucinations, and demonstrated strong retrieval performance. Further improvements could include tuning chunk size and reducing the number of retrieved documents to improve precision and answer correctness.
